01

O que é uma página web?

Toda página que você abre no navegador — Google, YouTube, Instagram — é feita com os mesmos ingredientes básicos. Não existe mágica. Existe código.

Mas esse código é dividido em responsabilidades muito claras. Pensa assim:

🏠 Analogia: A casa

Imagine que um site é uma casa.

• O HTML é a estrutura — paredes, teto, portas, janelas. Define o que existe e onde fica.

• O CSS é a decoração — cor da parede, tapete, cortinas, móveis. Define como tudo parece.

• O JavaScript é a elétrica e hidráulica — o que faz as coisas funcionarem: luz que acende, torneira que abre. (Veremos isso nas próximas aulas!)

💡 Por que aprender isso?
Você já usou dezenas de sites hoje. Depois desta aula, você vai conseguir criar um — e entender como tudo funciona por baixo do que você vê na tela.
📁
02

Os 3 arquivos do projeto

O projeto desta aula tem 3 arquivos na mesma pasta. Essa organização é a base de qualquer site real.

📁 minha-pagina/ ← pasta do projeto
├── 📄 index.html ← estrutura da página (o "esqueleto")
├── 🎨 style.css ← aparência visual (o "visual")
└── 🖼️ eu.jpg ← imagem usada na página
🧠 Conceito: Por que a pasta importa?
O HTML usa o nome do arquivo para encontrar recursos. Se o CSS está em outra pasta e você escrever só style.css, ele não vai achar. Todos os arquivos do projeto precisam estar juntos, ou você indica o caminho completo.
⚠️ Atenção: index.html é especial
Todo servidor web do mundo, quando alguém acessa um endereço sem especificar um arquivo (tipo 192.168.4.1), procura automaticamente por um arquivo chamado index.html. Por isso esse nome é padrão — é o ponto de entrada do site!
🦴
03

Estrutura do index.html

Todo arquivo HTML começa com a mesma "receita base". Veja o arquivo do projeto desta aula:

ARQUIVO DO ALUNO index.html 
<!DOCTYPE html>                  <!-- "Ei navegador, isso é HTML5!" -->
<html lang="pt-br">               <!-- idioma da página: português -->

  <head>                           <!-- cabeçalho: configurações invisíveis -->
    <meta charset="UTF-8">         <!-- aceita ç, ã, é, ó... -->
    <meta name="viewport" content="width=device-width">  <!-- funciona no celular -->
    <title>Minha primeira página</title>  <!-- texto da aba do navegador -->
    <link rel="stylesheet" href="style.css">  <!-- conecta o CSS! -->
  </head>

  <body>                           <!-- tudo que aparece na tela -->
    <h1>Olá, mundo!</h1>
    <img src="eu.jpg" alt="Exemplo de imagem">
    <p class="p1">Esta é minha primeira página HTML.</p>
    <p class="p2">Este é meu segundo parágrafo.</p>
    <p class="p3">Este é meu terceiro parágrafo.</p>
  </body>

</html>
🧅 Analogia: Tags são como cebolas

Tags HTML funcionam em camadas. A tag de fora "abraça" tudo que está dentro:

<html> abraça tudo → <head> e <body> ficam dentro → dentro do body ficam os textos, imagens, etc.

Isso se chama aninhamento (nesting). Nunca feche uma tag antes de fechar as que estão dentro dela!

TagO que éFica em
<!DOCTYPE html>Declaração de versão. Não é tag — é instrução para o navegador.Antes de tudo
<html lang="pt-br">Elemento raiz. Contém toda a página. lang define o idioma.Raiz
<head>Configurações da página. Nada aqui aparece na tela.Dentro de html
<body>Conteúdo visível. Tudo que o usuário vê e interage.Dentro de html

04

Tags essenciais do body

Dentro do <body>, usamos tags para dizer ao navegador o que é cada coisa — um título, um parágrafo, uma imagem, um vídeo.

TagO que fazExemplo do projeto
<h1> a <h6> Títulos. h1 é o mais importante (maior), h6 é o menor. <h1>Olá, mundo!</h1>
<p> Parágrafo de texto. O bloco mais comum de conteúdo. <p class="p1">...</p>
<img> Exibe uma imagem. Tag que se fecha sozinha (sem barra no final). <img src="eu.jpg" alt="...">
<video> Reproduz um vídeo. Precisa de <source> dentro. <video autoplay loop muted>
<a> Link clicável. O atributo href define o destino. <a href="pagina2.html">Ir</a>
🧠 Conceito: Atributos

Atributos são informações extras que você passa dentro da tag de abertura, no formato nome="valor".

Ex.: <img src="eu.jpg" alt="minha foto">

• src → "source" (fonte) — diz qual imagem carregar.

• alt → "alternative text" — texto que aparece se a imagem não carregar, e que ajuda leitores de tela para deficientes visuais.

Veja como o resultado aparece no navegador com o HTML puro (sem CSS ainda):

127.0.0.1 / index.html
SEM CSS

Olá, mundo!

Esta é minha primeira página HTML.

Este é meu segundo parágrafo.

Este é meu terceiro parágrafo.

↑ Sem CSS, o navegador usa seus estilos padrão. Funciona, mas é feio. Vamos melhorar isso!

05

O que é CSS e por que separar?

CSS (Cascading Style Sheets — Folhas de Estilo em Cascata) é a linguagem que define como os elementos HTML aparecem na tela: cor, tamanho, espaçamento, borda, fonte, posição.

👕 Analogia: Roupa e corpo

O HTML é o seu corpo — existente, com estrutura, com partes definidas (cabeça, braços, tronco).

O CSS é a roupa que você coloca. Você pode trocar a roupa (mudar o CSS) sem mudar o corpo (o HTML). O mesmo corpo pode ter aparências completamente diferentes só trocando as roupas!

Por que colocar o CSS em um arquivo separado (style.css) em vez de misturar com o HTML?

CSS junto no HTMLCSS em arquivo separado ✅
Arquivo bagunçado, difícil de ler. Código organizado e limpo.
Se quiser mudar a cor de todos os parágrafos, tem que achar um por um. Muda uma linha no CSS e todos os elementos mudam juntos.
Não funciona em várias páginas ao mesmo tempo. Um único style.css serve para quantas páginas você quiser.
HTML e visual misturados = difícil de manutenção. Separação de responsabilidades = profissional.
🧠 Conceito: Cascata
O "C" de CSS significa Cascata — os estilos "fluem" de cima para baixo como uma cachoeira. Um estilo definido no body vale para todos os elementos dentro dele, mas um estilo mais específico (numa .class, por exemplo) ganha do genérico. Isso é chamado de especificidade.
🔗
06

Ligando o HTML ao CSS

O HTML não sabe que o CSS existe a não ser que você avise explicitamente. Fazemos isso com a tag <link> dentro do <head>.

LINHA MAIS IMPORTANTE index.html — dentro do <head> 
<link rel="stylesheet" href="style.css">

<!--
  rel="stylesheet"  →  "esta é uma folha de estilo"
  href="style.css"  →  "encontre o arquivo chamado style.css"
                        (na mesma pasta que este index.html)
-->
⚠️ Erro clássico de iniciante

Esquecer de colocar a tag <link> ou errar o nome do arquivo (Style.css com S maiúsculo, por exemplo). Se o CSS não carregar, a página aparece sem estilo nenhum.

Dica: sempre salve o arquivo com letras minúsculas e sem espaços.

🔌 Analogia: A tomada
O HTML é o aparelho elétrico. O CSS é a energia. A tag <link> é o cabo que conecta os dois. Sem o cabo, o aparelho existe mas não funciona com sua capacidade total.
🎯
07

Seletores: tag, .class e #id

Para aplicar estilo, o CSS precisa saber quem vai receber aquele estilo. Para isso existem os seletores — formas de "apontar" para elementos específicos.

📄 HTML — o que existe
<h1>Olá, mundo!</h1>

<p class="p1">
  Parágrafo verde
</p>

<p class="p2">
  Parágrafo roxo
</p>

<p class="p3">
  Parágrafo amarelo
</p>
🎨 CSS — como vai parecer
/* seletor de TAG: afeta todo h1 */
h1 {
  color: red;
}

/* seletor de CLASS: afeta só .p1 */
.p1 { color: green; }

/* seletor de CLASS: afeta só .p2 */
.p2 { color: purple; }

/* seletor de CLASS: afeta só .p3 */
.p3 { color: yellow; }
SeletorComo escreverAfeta quem?Exemplo
Tag Só o nome da tag Todos os elementos daquela tag h1 { color: red; }
Class Ponto + nome da classe Todos os elementos com aquela class= .p1 { color: green; }
ID Hashtag + nome do id O elemento único com aquele id= #titulo { font-size: 2rem; }
Todos Asterisco * Absolutamente todos os elementos * { box-sizing: border-box; }
🧠 Conceito: class vs. id

• class → pode repetir. Vários elementos podem ter a mesma classe. Use para estilos que se repetem.

• id → deve ser único na página. Só um elemento pode ter aquele id. Use para elementos únicos (ex.: o cabeçalho principal, o botão de envio).

💡 Dica: Por que usar classes?
No projeto desta aula, os três parágrafos são todos <p> — a mesma tag. Se estilizarmos pela tag, os três ficariam iguais. Com classes diferentes (p1, p2, p3), podemos dar uma cor diferente a cada um!
⚙️
08

Propriedades CSS essenciais

Um bloco CSS tem sempre a mesma estrutura: seletor + chaves + propriedade: valor;

ARQUIVO DO ALUNO — COMPLETO style.css 
/* ─── Título ─────────────────────────────── */
h1 {
    color: red;        /* cor do texto */
}

/* ─── Parágrafos com classes diferentes ───── */
.p1 { color: green; }
.p2 { color: purple; }
.p3 { color: yellow; }

/* ─── Corpo da página ────────────────────── */
body {
    max-width: 800px;    /* largura máxima: não estica além de 800px */
    margin: 0 auto;      /* centraliza na tela (margem automática) */
    text-align: center;  /* centraliza o texto */
    color: blue;          /* cor padrão do texto */
}

/* ─── Imagem ─────────────────────────────── */
img {
    width: 200px;            /* largura fixa de 200 pixels */
    border: 5px solid black;  /* borda 5px, linha cheia, preta */
    display: block;           /* faz a imagem ser um bloco */
    margin: 0 auto;           /* centraliza a imagem */
    margin-bottom: 20px;      /* espaço embaixo da imagem */
}

/* ─── Vídeo ──────────────────────────────── */
video {
    display: block;       /* trata como bloco */
    margin: 0 auto;      /* centraliza */
    width: 100%;         /* ocupa a largura disponível */
    max-width: 800px;    /* mas nunca passa de 800px */
    margin-top: 20px;    /* espaço acima do vídeo */
}
🧠 Conceito: display block

Por padrão, <img> e <video> são elementos inline — ficam na mesma linha que o texto. Com display: block eles viram blocos, ocupam a linha toda sozinhos, e aí o margin: 0 auto consegue centralizá-los.

Regra: margin: 0 auto só centraliza elementos que são blocos (display: block) e que têm uma largura definida.

Propriedade CSSO que controlaExemplo do style.css
colorCor do textocolor: red — h1 vermelho
background-colorCor de fundo do elementobackground-color: #f0f0f0
font-sizeTamanho da fontefont-size: 18px
font-familyTipo de fonte (tipografia)font-family: Arial
text-alignAlinhamento do textotext-align: center
width / max-widthLargura do elementowidth: 200px — imagem
marginEspaço externo (fora da borda)margin: 0 auto — centraliza

09

Imagens e vídeo no HTML

Imagens e vídeos tornam uma página muito mais interessante. No HTML do projeto, temos os dois. Veja como funcionam:

IMAGEM E VÍDEO index.html 
<!-- ─── IMAGEM ──────────────────────────────── -->
<img
  src="eu.jpg"               <!-- src = nome do arquivo da imagem -->
  alt="Exemplo de imagem"   <!-- alt = descrição (acessibilidade) -->
>
<!-- <img> NÃO tem tag de fechamento! -->


<!-- ─── VÍDEO ───────────────────────────────── -->
<video
  autoplay          <!-- começa a tocar automaticamente -->
  loop              <!-- repete quando chega ao fim -->
  muted             <!-- sem som (obrigatório para autoplay funcionar) -->
  playsinline       <!-- toca no celular sem ir para tela cheia -->
>
  <source src="eu_gato_robo.mp4" type="video/mp4">
  Seu navegador não suporta vídeo.  <!-- texto de fallback -->
</video>
💡 Por que muted é obrigatório com autoplay?
Os navegadores modernos bloqueiam autoplay com som para não irritar o usuário. Se colocar autoplay sem muted, o vídeo simplesmente não vai iniciar. Com muted, o autoplay funciona normalmente.
🧠 Conceito: Formatos de arquivo

O navegador só consegue exibir tipos específicos de arquivo:

• Imagens: .jpg, .png, .gif, .webp, .svg

• Vídeos: .mp4 (H.264) funciona em todos os navegadores. Evite .avi, .mov e .wmv diretamente no HTML.

⚠️ Imagem não aparece? Veja esses erros

1. Nome errado: Eu.jpg ≠ eu.jpg (maiúscula/minúscula importa!)

2. Arquivo em outra pasta: se o HTML está na raiz e a imagem numa pasta imagens/, escreva src="imagens/eu.jpg"

3. Extensão errada: arquivo salvo como .jpeg mas escrito como .jpg no src.

Fichinha de referência + desafio

Guarde essas tags e propriedades — elas são os tijolos de qualquer página web:

<!DOCTYPE html>
Declaração obrigatória. Sempre a primeira linha.
<html lang="pt-br">
Elemento raiz. Envolve tudo.
<head>
Configurações invisíveis: charset, title, link CSS.
<body>
Tudo que aparece na tela.
<link rel="stylesheet" href="style.css">
Conecta o arquivo CSS ao HTML.
<h1> a <h6>
Títulos do mais importante ao menor.
<p class="nome">
Parágrafo com classe para estilizar.
<img src="arq.jpg" alt="desc">
Imagem. Tag sem fechamento.
<video autoplay loop muted>
Vídeo com reprodução automática.
color: valor
Cor do texto. Aceita nome ou código hex.
background-color: valor
Cor de fundo do elemento.
font-size: 18px
Tamanho do texto em pixels.
margin: 0 auto
Centraliza elemento bloco com largura definida.
border: 5px solid black
Borda: espessura, estilo, cor.
display: block
Transforma elemento em bloco.
width / max-width
Largura fixa ou máxima do elemento.
⚡ DESAFIO — Monte a sua página
  1. Crie uma pasta chamada minha-pagina no computador.
  2. Dentro dela crie dois arquivos: index.html e style.css.
  3. No HTML, adicione um <h1> com seu nome, uma foto sua (<img>) e 3 parágrafos sobre você — cada um com uma classe diferente.
  4. No CSS, dê uma cor diferente para cada parágrafo e coloque uma borda colorida na imagem.
  5. Bônus nível 1: use background-color no body para mudar a cor de fundo da página.
  6. Bônus nível 2: adicione font-size e font-family no body para mudar a fonte de toda a página.
  7. Bônus nível 3: pesquise "cores hexadecimais" e use códigos como #ff5733 em vez de nomes de cor.
